## Phase 1 — `env_loader`

import sys

| 플레이스홀더 키 | `your_upstage_api_key_here`면 `get_api_key()`가 None 또는 실패 처리 |

## Phase 2 — `chunking`

```python
# 프로젝트 루트에서: python scripts/test_chunking.py
import sys
from pathlib import Path
sys.path.insert(0, str(Path(__file__).resolve().parent.parent))

from src.chunking import FinancialDocumentChunker  # 클래스명은 구현에 맞게

pdf = "data/pdfs/your_sample.pdf"
c = FinancialDocumentChunker(chunk_size=1000, chunk_overlap=200)
chunks = c.process_pdf(pdf, chunking_method="langchain", enrich_metadata=False)

assert len(chunks) >= 1, "no chunks"
assert all("text" in ch and len(ch["text"]) > 20 for ch in chunks)
print("chunks:", len(chunks), "first 80 chars:", chunks[0]["text"][:80])
```

| 검증 | 성공 기준 |
|------|-----------|
| 청크 개수 | ≥ 1 |
| `enrich_metadata=False` | `MetadataFilter` 없이 동작 |
| `chunking_method="articles"` | 조항이 많은 PDF에서 청크 수·내용이 합리적 (선택 테스트) |

**실패 시 점검:** PDF가 스캔 이미지뿐이면 텍스트가 비어 있을 수 있음 → 다른 PDF 사용.

## Phase 3 — `embedding`

**주의:** 이 단계부터 API 호출(임베딩)이 발생해 **크레딧**이 사용됩니다.

```python
import os, sys
from pathlib import Path
sys.path.insert(0, str(Path(__file__).resolve().parent.parent))

from src.env_loader import load_environment
load_environment("config.env")

from src.chunking import FinancialDocumentChunker
from src.embedding import FinancialEmbeddingStore  # 구현 클래스명에 맞게

pdf = "data/pdfs/your_sample.pdf"
chunks = FinancialDocumentChunker(1000, 200).process_pdf(pdf, enrich_metadata=False)

store = FinancialEmbeddingStore(persist_directory="./vectordb_test", collection_name="test_rag")
store.reset_vectorstore()  # 구현에 있다면: 깨끗한 테스트
store.add_documents(chunks, metadata={"source": Path(pdf).name, "document_type": "test"})

hits = store.similarity_search("이자 금리 수수료 중 하나를 넣은 질문", k=3)
assert len(hits) >= 1
print("hit0:", hits[0].page_content[:120])
```

| 검증 | 성공 기준 |
|------|-----------|
| `add_documents` | 예외 없이 완료 |
| `similarity_search` | `Document` 리스트 길이 ≥ 1, `page_content` 비어 있지 않음 |
| `get_collection_info()` | `total_documents` ≥ 청크 수(또는 논리적으로 일치) |

**실패 시 점검:** `UPSTAGE_API_KEY`, 네트워크, `vectordb` 경로 권한, 컬렉션 이름 충돌.

## Phase 4 — `retriever`

```python
import sys
from pathlib import Path
sys.path.insert(0, str(Path(__file__).resolve().parent.parent))

from src.env_loader import load_environment
load_environment("config.env")

from src.embedding import FinancialEmbeddingStore
from src.retriever import FinancialDocumentRetriever

store = FinancialEmbeddingStore(persist_directory="./vectordb_test", collection_name="test_rag")
assert store.load_existing_vectorstore(), "index the PDF first (Phase 3 test)"

r = FinancialDocumentRetriever(store)
out = r.answer_question("이 상품의 이자율은 어떻게 되나요?")  # PDF 내용에 맞게 질문 수정

assert "answer" in out and len(out["answer"]) > 5
assert out.get("sources"), "expected sources"
print(out["answer"][:200])
```

| 검증 | 성공 기준 |
|------|-----------|
| `answer` | 질문과 관련된 한글 문장 (환각이면 프롬프트·k값·청크 품질 점검) |
| `sources` | 리스트 길이 ≥ 1 |

## Phase 5 — `metadata_filter` (선택)

- `process_pdf(..., enrich_metadata=True)` 후 Phase 3·4 테스트를 **동일하게** 반복  
- Chroma `similarity_search(..., filter={...})`를 쓰는 경우, 필터 조건에 맞는 문서만 나오는지 확인

## Phase 6 — Streamlit (`app.py`)

| 단계 | 사용자 동작 | 성공 기준 |
|------|-------------|-----------|
| 1 | 사이드바에서 API 키 입력·저장 | 메인 화면 진행 가능 |
| 2 | PDF 업로드 → 처리·벡터화 | 성공 메시지, 청크 수 표시 |
| 3 | 질문 입력 → 질문하기 | 답변 영역에 텍스트 |
| 4 | 참고 문서 expander | 청크 내용·메타데이터 표시 |
| 5 | Ctrl+C (Mac) | 서버 종료 (`run.sh`는 `exec streamlit` 권장) |

**자동화가 어렵다면** 위 표를 수동 체크리스트로만 사용해도 됩니다.

## 문제 빠른 매핑

| 증상 | 우선 확인 |
|------|-----------|
| `UPSTAGE_API_KEY` 오류 | `config.env`, Streamlit 세션, `load_environment` 호출 시점 |
| Chroma empty / 검색 0건 | 인덱싱이 끝났는지, 컬렉션 이름·경로 일치 |
| 임베딩 HTTP 오류 | 키 만료·쿼터·모델명 문자열 |
| 청크 0개 | PDF 텍스트 추출 실패(스캔 PDF) |
| `MetadataFilter` import 오류 | `enrich_metadata=False`로 두거나 Phase 5 완료 후 True |
